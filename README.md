# Database-Project
Group-6 Database project(Room-management).
Overview
The Room Management System (RMS) is a relational database project designed to streamline hotel operations. It manages guests, bookings, rooms, staff, housekeeping, and maintenance activities in one integrated system. The project demonstrates how database theory can be applied to solve real-world hotel management challenges.

Features
Guest Management: Store guest details including contact information and nationality.

Room Management: Track room types, status, and capacity.

Bookings: Handle reservations, check-in/check-out dates, and payment status.

Staff Management: Maintain staff roles, shifts, and contact details.

Housekeeping & Maintenance: Schedule tasks, log issues, and track completion.

Reports: Generate queries for occupancy, revenue, staff distribution, and room status.

Database Design
Entities: Guests, Bookings, Room, Room_Type, Room_Status, Staff, Staff_Role, Housekeeping, Maintnance.

Relationships: One-to-many links between room types and rooms, guests and bookings, staff and housekeeping/maintenance tasks.

ER Diagram: Included in the project report to visualize relationships.

Implementation
Built using MySQL.

Tables created with proper primary keys, foreign keys, and constraints.

Populated with realistic sample data (100 rooms, guests, bookings, staff, housekeeping, and maintenance records).

Queries
Sample queries included in the project:

Room Status Overview

Staff by Role

Staff Role Permissions

Room Type Summary

Active vs Inactive Staff

Staff Contact List

Each query demonstrates how the system supports hotel operations and management decision-making.

How to Run
Import the database schema into MySQL.

Load the sample data provided.

Run the SQL queries to generate reports.

Review outputs to validate system functionality.
